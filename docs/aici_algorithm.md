# AICI 算法白皮书：AI 协同指数多信号融合画像

## 1. 设计动机

在 Python 实验教学场景中，传统「AIGC 检测 / AI 写没写」的二元判断既不准确也无教育意义——课程本身就要求学生学习 `import`、函数、类等结构，这些「AI 特征」恰恰是教学目标。因此本系统放弃二元判定，改为 **AI 协同指数（AI Collaboration Index, AICI）**：输出 0–100 的连续指数与四级标签（自主 / 协同 / 依赖 / 疑似代劳），用途是**帮助教师定位需要个别化辅导的学生，而非用于惩戒**。

设计原则：把「检测不可靠」这一缺陷重新表述为更教育学的目标——我们不做不可靠的代笔判定，而做可解释的 AI 协同画像。

## 2. 信号体系与权重

| 信号 | 数据源（系统已有） | 特征 | 权重 |
|---|---|---|---|
| ① 过程行为 | `submissions.created_at/submitted_at` + `tutor_chats` | 完成时长、零对话高分、极速高分 | 25% |
| ② 对话语义 | `tutor_chats.content` + `ANSWER_SEEK_KEYWORDS`（23 词表） | 索取答案次数、消息平均长度 | 20% |
| ③ 代码风格 | `ai_collab_final_code` + 同实验班级基线 | 注释密度/main 守卫/类型注解/英文注释比的 Z 分数（仅取「过度规范」方向） | 25% |
| ④ 答辩验证 | `defense_score` vs `total_score` | 作品高分但答辩低分的落差 | 20% |
| ⑤ 元信息 | 粘贴长度 / 击键节奏（需新埋点） | 单次粘贴分布、击键间隔方差 | 10%（暂缺省） |

第 ⑤ 类元信息需新增前端埋点，当前版本未启用；其余四类权重在融合时**归一化到 1.0**，使指数用满 0–100 量程。

## 3. 计算流程

1. 对每个实验，先用该实验全部已完成提交计算代码风格基线（各特征的均值与标准差）。
2. 对每条提交分别计算四类信号子分（均为 0–100）。
3. 加权归一化融合得到 AICI，并按阈值映射四级标签：自主 `<30` / 协同 `30–60` / 依赖 `60–80` / 疑似代劳 `≥80`。
4. 结果与四个分量明细一并写入 `aigc_detections.detection_detail`（JSON），教师端可下钻查看「为什么是这个分」。

代码实现见 `aici.py`（纯函数，线上 Flask 路由与离线批量回算脚本 `backfill_aici.py` 复用同一套口径），线上入口见 `app.py` 的 `/api/aigc/detect` 与教师批量回算 `/api/aici/recompute/<exp_id>`。

## 4. 在真实数据上的回算结果

对系统中 **597 条已完成提交**全量回算（`backfill_aici.py`）：

- 等级分布：自主 592、协同 5（无「依赖 / 疑似代劳」）。
- 指数分布：最小 0、均值 5.8、最高 36.1。
- 各信号触发样本数：过程行为 61、对话语义 129、代码风格 378、答辩验证 0（答辩管线为试点阶段，落差信号尚未参与）。

**解读**：该批次学生整体属于正当协同使用 AI，指数集中在低区间，仅在少量样本上触发单信号提示。这正是教学 triage 期望的形态——不把多数学生误标为「AI 依赖」，而是把教师注意力引导到相对高信号的尾部。指数的主要价值在**相对排序与下钻复核**，而非绝对阈值告警。随着答辩管线全面上线，第 ④ 类信号将补齐，对「作品高分但答辩低分」的疑似代劳样本识别力会显著提升。

## 5. 局限与下一步

- 单信号难以单独定性，必须结合教师人工复核；系统定位为「辅助画像」而非「自动裁决」。
- 第 ⑤ 类元信息埋点未上线，期末版本将补充粘贴/击键行为采集。
- 代码风格基线对小样本实验（如第 8 次实验仅 28 份提交）稳健性有限，已用 `stdev` 兜底处理。
