# 面向 Python 实验教学的生成式 AI 助教与过程性评价系统

> 一线教师自主开发、接入国产大模型的 Python 程序设计实验教学**智能信息系统**：把生成式 AI 从"替学生完成任务的工具"转化为"支撑教师诊断、学生反思与过程治理"的过程性评价系统。

本项目为 **2026 年教师人工智能应用案例（创 AI · 智能信息系统）** 配套开源发布版（脱敏）。系统已在高校 Python 实验课真实运行约两个月，覆盖 7 个班级 128 名学生。

---

## 解决的教学问题

Python 实验课学生基础差异大，商业大模型被当作"答案生成器"导致 **AI 代写、答案索取、学习过程黑箱**，教师难以及时辅导、诊断与评价学生真实能力。本系统围绕"目标可测、过程可见、能力可评、风险可控"重建实验教学闭环。

## 核心特性

- **四阶段过程性评价**：每次实验拆为概念理解 / AI 协同编程 / 程序分析 / 综合应用四阶段（25+30+25+20 分），形成"作品—交互—答辩—复核"学习证据链。
- **苏格拉底式 AI 助教**：不直接给可复制答案，通过追问与小步引导促学生自行得出结论。
- **AI 协同画像（AICI）**：多信号融合（过程行为、对话语义、代码风格、答辩落差）输出 0–100 协同指数与"自主 / 协同 / 依赖 / 疑似代劳"四级标签。
- **课前智能答辩**：基于上次提交自动生成个性化闭卷题（理解验证 / 迁移应用 / AI 协同思考），检验真实理解。
- **多规则早期预警**：8 条规则识别需关注学生（阶段超时、高分零对话、直接索取答案等）。
- **三层削峰填谷 + 混合云调度**：题库预生成 + 异步评分队列 + 实时辅导队列，配合本地 / 云端混合调度与预算溢出，应对课堂高峰。
- **数据安全与治理**：最小必要采集、信息脱敏、权限控制、人工复核；以本地或混合云模式运行时学生过程数据"数据不出校"。

## 技术栈

Python 3 · Flask · SQLAlchemy · SQLite · 原生 HTML+JS · 行业标准（OpenAI 兼容）推理接口

## 快速开始

```bash
pip install flask flask-sqlalchemy
# 配置推理端点（二选一）
export LLM_API_URL="https://<国产云端大模型 API>/v1/chat/completions"
export LLM_API_KEY="<你的 API Key>"
python migrate_and_seed.py   # 初始化数据库与演示数据
python app.py                # 启动，默认 http://127.0.0.1:5000
```

**软硬件门槛低**：无需自建本地 AI 算力——在教师端"模型配置"页填入一个国产云端大模型 API Key 即可接入运行，普通云服务器即可部署，兄弟院校可零门槛复用；如需"数据不出校"，再切换本地 / 混合云模式（对接昇腾等私有化推理端点）。

## 数据与隐私

本仓库为**脱敏开源版**：不含真实学生数据库、不含任何 API 密钥与服务器配置。`instance/` 数据库与 `.env` 已通过 `.gitignore` 排除；运行所需密钥请通过环境变量注入，请勿硬编码。
